# Project Review

## 触发条件

- 关键任务主产出完成后
- 需要输出统一项目级 review 结论时

## 核心原则

- review 结论必须明确是否允许推进
- review 必须可回写正式文档、连续性资产或任务机结果

## 必须先读的文档

1. `doc/01-规范标准/文档治理/AI阅读路径规范/current/开发规范.md`
2. `doc/01-规范标准/文档治理/AI阅读路径规范/current/问题修复文档归档与执行规范.md`

## 任务分型 / 判断分支

1. `doc-review`
2. `code-review`
3. `release-or-task-machine-review`

## 最小检查清单

- 是否有遗漏项
- 是否有冲突项
- 是否有风险项
- 是否必须补文档 / 门禁 / 交接

## 输出模板或结论格式

- 复核对象
- 是否通过
- 遗漏项
- 冲突项
- 风险项
- 必须补的文档 / 门禁 / 交接
- 是否允许推进
