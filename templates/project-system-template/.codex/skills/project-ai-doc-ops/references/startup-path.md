# Startup Path

Use this order when entering a substantive repository task:

1. [CLAUDE.md](__PROJECT_ROOT__/CLAUDE.md)
2. [doc/README.md](__PROJECT_ROOT__/doc/README.md)
3. [doc/00-入口/README.md](__PROJECT_ROOT__/doc/00-入口/README.md)
4. [doc/00-入口/AI执行入口.md](__PROJECT_ROOT__/doc/00-入口/AI执行入口.md)
5. [doc/00-入口/当前生效版本说明.md](__PROJECT_ROOT__/doc/00-入口/当前生效版本说明.md)

Then narrow by task:

- UI / frontend work: read the current frontend standards or version frontend docs
- Backend / database / API work: read the current backend standards or version backend / database docs
- Deployment / testing / troubleshooting: read the matching SOP or problem-summary docs
- Formal document work: load [project-doc-governance](__PROJECT_ROOT__/.codex/skills/project-doc-governance/SKILL.md)

If the task enters code work, also read:

1. [开发规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/开发规范.md)
2. the current approved implementation document
