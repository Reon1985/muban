# 项目专属 Skills

## 1. 目录定位

本目录存放本项目专属的 skills，用于：

- 强化项目核心规则的执行
- 提供项目特定场景的辅助流程
- 与全局 agent / skill 配合使用

## 2. 当前 Skills 清单

| Skill | 用途 | 触发场景 |
|-------|------|----------|
| [需求审查前置](./需求审查前置/SKILL.md) | 用户提出修改意见时的前置审查 | 需求/页面/交互/范围变更 |
| [实施门禁检查](./实施门禁检查/SKILL.md) | 确认是否可进入开发阶段 | 进入正式开发前 |
| [UI阶段确认](./UI阶段确认/SKILL.md) | UI 阶段高频调整与确认 | 模块 UI 确认阶段 |
| [知识回填提醒](./知识回填提醒/SKILL.md) | 探索新结论后的知识沉淀 | 发现新框架认知/规范 |
| [效率复盘](./效率复盘/SKILL.md) | 开发超过 8 分钟后的复盘 | 正式开发收尾时 |
| [项目导航](./项目导航/SKILL.md) | 快速定位文档和代码 | 需要查找文档/代码 |

## 3. 与全局 Skills 的关系

### 项目专属 Skills

负责：

- 项目特定流程的执行
- 项目核心规则的落地
- 项目文档体系的维护

### 全局 Skills

负责：

- 通用开发能力（`agency-*`）
- 通用流程能力（`gstack-*`）
- 系统基础能力（`.system/*`）

### 配合使用方式

推荐顺序：

1. 先用项目专属 skill 明确流程和规则
2. 再用全局 `agency-*` 选择专业角色
3. 再用全局 `gstack-*` 选择流程技能
4. 最后用项目专属 skill 完成复盘和回填

典型组合：

```text
需求审查前置（项目专属）
→ agency-product-manager（全局）
→ 实施门禁检查（项目专属）
→ agency-frontend-developer（全局）
→ gstack-review（全局）
→ 效率复盘（项目专属）
→ 知识回填提醒（项目专属）
```

## 4. 使用建议

### 开发任务

```text
项目导航 → 实施门禁检查 → [全局 agent] → 效率复盘 → 知识回填提醒
```

### 需求修改

```text
需求审查前置 → [全局 agent] → UI阶段确认 → 知识回填提醒
```

### 探索任务

```text
项目导航 → [探索] → 知识回填提醒
```

## 5. 维护规则

- 新增 skill 后，更新本 README 清单
- Skill 内容变化后，同步更新 SKILL.md
- Skill 废弃后，从本 README 删除并保留历史记录
- 与全局 skill 有重叠或冲突时，优先使用项目专属 skill
