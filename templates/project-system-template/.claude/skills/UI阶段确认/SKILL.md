# UI 阶段确认

## 触发条件

当任务处于模块开发的"UI 确认阶段"时触发。

用于：

- 高频 UI 微调
- UI 确认收口
- 从 UI 阶段切换到后端阶段的决策

## 核心原则

**先把页面和交互看对，再做该模块后端。**

UI 未确认前，不进入该模块的数据库表结构、接口契约和业务服务正式开发。

## UI 阶段规则

### 允许高频调整

在同一份已批准实施文档覆盖的 UI 范围内，可连续做多轮前端微调。

允许范围：

- 页面布局微调
- 组件样式微调
- 文案微调
- 交互细节微调

### 不允许擅自扩大

以下改动一旦发生，必须回到实施文档重新确认：

- 菜单结构调整
- 页面规则变化
- 交互流程变化
- 实现范围变化
- 模块边界变化

### 调整分类判断

每次 UI 微调时，先判断：

```markdown
## 本次调整类型判断

### 调整内容
- [描述]

### 类型判断
- [ ] 高频 UI 微调（允许连续推进）
- [ ] 页面规则变化（需重新确认）
- [ ] 交互流程变化（需重新确认）
- [ ] 实现范围变化（需重新确认）

### 判断依据
- [依据说明]

### 建议动作
- [直接调整 / 先更新实施文档]
```

## UI 确认收口

### 确认标准

UI 确认需满足：

- [ ] 页面结构符合需求
- [ ] 菜单导航符合需求
- [ ] 交互流程符合需求
- [ ] 视觉规范符合 UI 规范文档
- [ ] 用户已确认页面效果

### 确认记录模板

```markdown
## UI 确认记录

### 确认时间
- [时间]

### 确认范围
- 模块：[模块名称]
- 页面：[页面清单]
- 实施文档：[文档名称及版本]

### 确认结论
- 页面结构：[符合/需调整]
- 菜单导航：[符合/需调整]
- 交互流程：[符合/需调整]
- 视觉规范：[符合/需调整]
- 用户确认：[已确认/未确认]

### 确认后的锁定项
- [锁定内容清单]

### 确认后的待后续项
- [待后续内容清单]

### 后端阶段准备
- [可进入后端阶段 / 需补充确认]
```

## 从 UI 阶段切换到后端阶段

### 切换前检查

切换到后端阶段前，必须确认：

- [ ] UI 已完成用户确认
- [ ] 实施文档已明确后端范围
- [ ] 菜单结构、页面规则、交互流程已锁定
- [ ] 后端开发将以已确认 UI 为依据

### 切换动作

确认后：

1. 更新当前模块的阶段状态为"后端贯通阶段"
2. 明确后端开发范围和边界
3. 如需更新实施文档，先完成更新
4. 进入后端开发

### 切换记录模板

```markdown
## UI → 后端阶段切换记录

### 切换时间
- [时间]

### 切换模块
- [模块名称]

### UI 确认依据
- 实施文档：[名称及版本]
- UI 确认记录：[记录位置]

### 后端开发范围
- 数据库表：[如有]
- 接口契约：[如有]
- 业务服务：[如有]

### 后端开发边界
- 必须以已确认 UI 为依据
- 不反向重做页面
- 不超出已批准实施文档范围

### 审查 agent
- 主开发 agent：[名称]
- 审查 agent：[名称]
```

## 注意事项

- UI 阶段的核心目标是"先把页面看对"
- 高频微调是允许的，但要守住实施边界
- 一旦 UI 确认，后端必须以已确认 UI 为依据
- 不要因为后端开发时发现新问题，又反向推翻 UI 确认
