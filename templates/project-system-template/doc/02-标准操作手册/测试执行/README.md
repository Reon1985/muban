## 2. 使用原则

1. 跨版本复用的测试账号、组织、部门、客户、员工基线，统一维护在本目录。
2. 某次测试报告如果实际使用了这套基线数据，应直接引用本目录文档，不再在每份报告中重复抄录整套账号表。
3. 如果后续新增了专门用于权限差异、跨财务组织隔离、禁用态验证的账号，也继续补充到同一份基线文档中。
4. 如果本轮修复明确来源于测试报告，默认先按“测试报告批量收口与统一发版验证”卡片冻结问题清单，再进入发版。
