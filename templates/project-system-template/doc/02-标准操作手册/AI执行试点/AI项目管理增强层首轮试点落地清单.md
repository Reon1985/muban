# AI项目管理增强层首轮试点落地清单

本文件用于指导 `AI项目管理增强层 6 模块整体实施方案` 的首轮试点执行。

本文件是执行清单，不是长期原则文档。

4. [项目级知识隔离规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/项目级知识隔离规范.md)

## 2. 试点目标

首轮试点只验证以下 `4` 件事：

1. 复杂任务蓝图能否稳定指导一轮真实任务推进
2. 新项目接入规范能否支撑一个新仓库建立最小骨架
3. 项目级知识隔离规则能否阻断跨项目事实污染
4. 轻量自动学习能否只形成候选经验，而不越界成自动进化

本轮不验证：

1. 重型自动化 hook
2. 自动改 skill
3. 自动改规则
4. 自动改门禁

---

## 3. 试点对象

建议选择以下 `3` 类对象作为试点：

1. 当前 `__PROJECT_NAME__` 仓库中的一个真实复杂任务
2. 一个需要迁移相同方法论的新项目
3. 一轮真实任务收尾后的候选经验池整理

---

## 4. 试点步骤

### 4.1 蓝图试点

1. 选定一个真实复杂任务
2. 按 [复杂任务蓝图规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/复杂任务蓝图规范.md) 建立阶段清单
3. 标出每阶段输入、输出、门禁和复核点
4. 记录是否出现漏项、跳步或返工

### 4.2 接入试点

1. 选定一个新项目或模拟接入场景
2. 按 [新项目接入规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/新项目接入规范.md) 检查入口、分层和版本口径
3. 记录哪些项目事实需要重建，哪些方法可以复用

### 4.3 隔离试点

1. 选定一次跨项目迁移或跨仓库复用场景
2. 按 [项目级知识隔离规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/项目级知识隔离规范.md) 区分项目事实、通用方法和升级候选
3. 记录是否成功阻断事实误迁移

### 4.4 自动学习试点

1. 在任务收尾时整理一轮候选经验池
2. 只记录候选，不直接改正式规则
3. 判断哪些候选应回填到规范、SOP、问题总结或升级日志

---

## 5. 试点产出

首轮试点完成后，至少要产出：

1. 一份复杂任务蓝图样例
2. 一份新项目接入检查结果
3. 一份跨项目隔离判断样例
4. 一份候选经验池样例
5. 一份试点复盘结论

---

## 6. 通过标准

只有以下条件满足，才建议进入下一阶段：

1. 三份骨架规则都能指导真实场景，而不是停留在抽象描述
2. 试点过程没有出现明显重复、冲突或不可执行项
3. 自动学习候选没有越界进入自动进化
4. 试点发现的问题已经有明确回填落点

---

## 7. 试点后的回填要求

试点结束后，必须至少完成：

1. 把稳定结论回填到 `01-规范标准/` 或 `02-标准操作手册/`
2. 把单次问题回填到 `04-问题总结/`
3. 把 AI 执行体系本身的升级回填到 [AI能力与规则升级日志.md](__PROJECT_ROOT__/doc/05-升级日志/AI执行体系/AI能力与规则升级日志.md)
