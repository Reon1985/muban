# 轻量自动学习与自动进化边界规范

## 1. 文档定位

本文件用于将“轻量自动学习”正式纳入当前项目的 AI 执行体系，并明确它与“自动进化”的边界。

本文件重点回答：

1. 什么属于轻量自动学习
2. 什么已经越界为自动进化
3. 当前项目为什么只允许前者，不允许后者默认生效
4. 候选经验应该如何进入正式文档体系

边界说明：

1. 本文件负责“自动学习 / 自动进化”的概念边界与治理规则
2. [WORKFLOW.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/WORKFLOW.md) 负责说明它在任务收尾与复盘中的位置
3. [开发规范.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/AI阅读路径规范/current/开发规范.md) 负责说明开发阶段如何收集候选经验、如何回填
4. [AI能力与规则升级日志.md](__PROJECT_ROOT__/doc/05-升级日志/AI执行体系/AI能力与规则升级日志.md) 负责记录本机制后续升级

## 2. 当前结论

当前项目允许：

1. 轻量自动学习

当前项目默认不允许：

1. 自动进化直接生效

一句话规则：

- 机器可以自动收集候选经验，但不能自动改变正式规则、正式流程、skill 或 hook。

---

## 3. 轻量自动学习的定义

轻量自动学习，是指 AI 在任务执行过程中，自动收集“可能值得沉淀”的候选经验，但不自动让其成为正式规则。

它的目标是：

1. 降低重复踩坑
2. 提高复盘效率
3. 更早发现值得长期化的规则
4. 帮助主线程与复核 agent 更快判断“这次是否要回填”

允许的自动学习动作包括：

1. 自动收集错误和失败点
2. 自动收集用户纠正过的错误理解
3. 自动收集高频重复问题
4. 自动生成文档回填候选项
5. 自动生成 SOP 候选项
6. 自动生成 hook 候选项
7. 自动生成规则优化候选项

这些动作的共同特征是：

1. 只收集
2. 只提示
3. 只形成候选池
4. 不直接生效

---

## 4. 自动进化的定义

自动进化，是指 AI 不只记录经验，而是根据经验自动改变自身执行体系。

典型表现包括：

1. 自动修改正式规范文档
2. 自动修改 skill 正文
3. 自动调整默认工作流
4. 自动新增或升级阻断类 hook
5. 自动改变阶段门禁
6. 自动改变 agent / review 机制

这些动作的共同特征是：

1. 不只是记录经验
2. 而是直接改变未来默认执行方式

当前项目默认不允许上述动作自动生效。

---

## 5. 当前项目的边界规则

### 5.1 允许做的事

AI 允许自动做以下事情：

1. 在任务结束时汇总“本轮经验候选”
2. 在效率复盘时标记“高频重复问题”
3. 在回填前给出“建议补到哪里”的候选列表
4. 在规则审查时提示“这条经验可能值得长期化”

### 5.2 不允许自动做的事

AI 默认不允许自动做以下事情：

1. 把候选经验直接写成正式规范并默认生效
2. 未经确认自动修改 skill
3. 未经确认自动新增默认 hook
4. 未经确认自动把经验升级成新的门禁
5. 未经确认自动改变默认 agent 选型或复核机制

### 5.3 判断分界线

判断某个动作属于“轻量自动学习”还是“自动进化”，只看一条：

- 是否自动改变了正式执行规则

如果答案是“是”，则该动作属于自动进化。

---

## 6. 当前项目的标准模式

当前项目统一采用以下模式：

```text
自动收集候选经验
→ 人工或正式流程审查
→ 回填正式文档
→ 如有必要，再人工升级为 skill / hook / SOP / 长期规则
```

一句话：

- 机器收集，人工裁决。

---

## 7. 候选经验的推荐分类

任务执行过程中，自动学习候选建议统一按以下类型整理：

1. 错误候选
   - 本轮出现的真实错误、失败点、越界动作
2. 规则候选
   - 可能值得进入长期规范的执行规则
3. SOP 候选
   - 可能值得进入标准操作手册的高频动作
4. hook 候选
   - 可能适合自动提醒、自动检查的动作
5. 风险候选
   - 未来跨项目、跨阶段、跨环境可能重复触发的风险点

---

## 8. 正式回填口径

如果候选经验经审查成立，必须按类型回填：

1. 长期 AI / 规则 / skill 治理规则：
   - 回填到 `01-规范标准/文档治理/...`
2. 高频执行动作：
   - 回填到 `02-标准操作手册/...`
3. 单次踩坑、限制或排障结论：
   - 回填到 `04-问题总结/...`
4. AI 执行机制本身的升级：
   - 回填到 [AI能力与规则升级日志.md](__PROJECT_ROOT__/doc/05-升级日志/AI执行体系/AI能力与规则升级日志.md)

禁止直接回填到：

1. 临时会话记录
2. skill 内部私有注释
3. 未确认的待办片段

---

## 9. 多项目隔离口径

轻量自动学习必须遵守项目级知识隔离规则：

1. 一个项目中自动收集到的候选经验，默认只在该项目内有效
2. 不允许默认跨项目生效
3. 只有被明确抽象为“通用方法论”后，才允许升级到全局层

---

## 10. 当前执行建议

当前项目建议先采用最轻版本：

1. 在任务收尾或复盘阶段自动整理“经验候选池”
2. 候选池只做提示，不自动改规则
3. 由主线程或正式评审决定是否升级为长期规则、SOP、hook 或 skill

当前不建议直接上：

1. 自动改 skill
2. 自动改 hook
3. 自动改门禁
4. 自动改默认流程

---

## 11. 一句话结论

- 轻量自动学习可以进入当前体系，但只能作为候选经验收集机制存在，不能默认演化为自动进化机制。
