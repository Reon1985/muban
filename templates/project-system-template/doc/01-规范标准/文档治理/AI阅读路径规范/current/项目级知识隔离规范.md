# 项目级知识隔离规范

## 1. 文档定位

本文件用于定义多项目并行时，哪些内容可以迁移，哪些内容必须留在项目内，不得默认外溢。

本文件回答的是“多项目并行时如何不串味”，不负责替代业务需求、数据库设计或版本实施边界。

## 2. 核心原则

默认只允许迁移“方法”，不允许默认迁移“事实”。

必须区分以下三类内容：

1. 项目事实
   - 当前项目的版本、环境、账号、地址、路径、组织结构、业务口径
2. 通用方法
   - 可复用的流程、模板、检查项、治理规则
3. 升级候选
   - 在某项目中被验证过，但还未被正式抽象为通用方法的经验

---

## 3. 默认隔离规则

### 3.1 项目事实默认只在本项目内有效

以下内容默认禁止跨项目直接复用：

1. 服务器地址
2. 数据库表真实口径
3. 具体页面路径
4. 账号密码
5. 项目专属业务规则
6. 当前版本边界

### 3.2 通用方法允许跨项目迁移

以下内容可以在抽象后跨项目使用：

1. 文档结构方法
2. 复杂任务推进骨架
3. 复核机制
4. 预览优先方法
5. 回填与日志方法

### 3.3 升级候选不能默认升级为通用规则

以下内容即使在当前项目有效，也只能先作为候选项：

1. 新发现的执行经验
2. 高效但尚未标准化的工作法
3. 一次性排障结论

---

## 4. 跨项目迁移标准动作

当需要把一个项目中的经验迁移到另一个项目时，必须按以下顺序处理：

```text
识别原始内容
→ 判断属于项目事实 / 通用方法 / 升级候选
→ 仅抽象通用方法
→ 在目标项目中重新确认事实
→ 必要时再回填目标项目文档
```

执行要求：

1. 目标项目中的事实必须重新确认，不得沿用源项目事实
2. 迁移时必须保留来源说明，避免误以为是目标项目既有事实

---

## 5. 与自动学习的关系

轻量自动学习收集到的候选经验，默认只在当前项目中有效。

只有满足以下条件，才允许考虑升级为通用方法：

1. 已在当前项目稳定验证
2. 已与项目事实脱钩
3. 已形成可复用的抽象表述
4. 已经过正式审查和回填

---

## 6. 默认禁止项

默认禁止以下行为：

1. 把一个项目的版本入口直接复制成另一个项目的默认入口
2. 把一个项目中的环境配置、路径和账号当作通用规则
3. 把候选经验未经审查直接升级到全局层
4. 在未重新确认事实前，直接复用历史会话结论

---

## 7. 完成标准

项目级知识隔离规则建立完成，至少应满足：

1. 项目事实、通用方法、升级候选三类边界已明确
2. 跨项目迁移有统一动作顺序
3. 自动学习结果不会默认跨项目生效
4. 后续治理能清楚判断某条经验应留在项目内还是可抽象升级
