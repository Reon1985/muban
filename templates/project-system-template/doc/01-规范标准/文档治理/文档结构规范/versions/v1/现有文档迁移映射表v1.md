# 现有文档迁移映射表 v1

## 1. 文档定位

本文件用于承接 [项目文档体系整体重构方案v1.md](__PROJECT_ROOT__/doc/01-规范标准/文档治理/文档结构规范/versions/v1/项目文档体系整体重构方案v1.md) 的下一步落地工作，回答下面三个问题：

1. 当前仓库中的各类正式文档，未来应迁移到哪里
2. 哪些文档可以直接保留，哪些需要拆分、合并、改名或归档
3. 哪些文档目录在迁移后将不再保留原有职责边界

本文件是**迁移设计映射表**，不是执行记录。后续真正开始目录改造时，应以本文件作为迁移清单基础。

---

## 2. 映射动作定义

后续迁移统一采用以下动作标识：

| 动作 | 含义 | 说明 |
| --- | --- | --- |
| `M` | 直接迁移 | 文档主体可保留，主要做移动、改名、补索引 |
| `S` | 拆分后迁移 | 一份文档包含多个层级内容，需要拆成多份再迁移 |
| `C` | 合并重写 | 多份同类文档建议合并成新的权威文档 |
| `A` | 归档下沉 | 迁移后不再作为正式入口，只保留历史参考价值 |
| `D` | 废弃替换 | 旧文档失去正式职责，由新文档替代 |

---

## 3. 新结构目标引用

本映射表统一以以下目标结构为准：

```text
doc/
├── 00-入口/
├── 01-规范标准/
├── 02-标准操作手册/
├── 03-版本包/
├── 04-问题总结/
├── 05-升级日志/
└── 99-参考与归档/
```

补充说明：

1. `UI规范` 属于 `01-规范标准/前端/UI规范/`
2. 业务版本内页面方案属于 `03-版本包/<版本>/前端/`
3. 部署、发版、测试等高频动作的执行入口属于 `02-标准操作手册/`
4. 历史输入件、旧版映射表、探索过程稿统一进入 `99-参考与归档/`
