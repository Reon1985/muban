# 项目任务机运行资产与拦截规范

## 1. 文档定位

本文件定义任务机、连续性状态和交接资产在项目中的使用方式。

## 2. 启用原则

1. 正式文档入口链仍是首要权威来源。
2. TraceID、`.project_state.json`、`.handoff/` 用于补足续接、交接、回填和审计链路。
3. 如果项目规模较小、任务较短或团队没有这类需要，可以不把这套机制作为硬前置。

## 3. 默认检查项

1. 当前任务是否已有 TraceID。
2. 跨会话任务是否已有 handoff。
3. 当前活跃任务是否已登记主执行者。
4. 关键产出是否已有独立复核者与复核结论。
5. 关键任务收尾时是否已回写状态、下一步建议和必要文档回填。

## 4. 拦截口径

启动前不应直接进入正式推进的情况：

1. 已启用任务机的关键任务缺少 TraceID。
2. 已启用任务机的关键任务缺少主执行者记录。
3. 已启用任务机的跨会话任务缺少 handoff。

完成前不应直接标记收口的情况：

1. 已启用任务机的关键产出缺少独立复核者。
2. 已启用任务机的关键产出缺少独立复核结论。
3. 已启用任务机的关键任务缺少必要文档回填或下一步交接结论。
