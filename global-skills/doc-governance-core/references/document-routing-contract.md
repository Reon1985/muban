# Document Routing Contract

The core skill assumes every repository has an approved formal-document structure, but it does not dictate one universal folder tree.

The project adapter should define:

1. where entry and navigation docs live
2. where long-lived standards live
3. where SOP or runbook docs live
4. where version-specific deliverables live
5. where problem summaries and troubleshooting docs live
6. where upgrade logs and execution evidence live
7. where raw references and archives live
8. naming rules and fixed-name governance docs

The adapter should also state:

- whether legacy compatibility folders still exist
- whether migration is still in progress
- which folders are authoritative for new formal outputs

When moving the workflow into a new project:

- keep this core skill unchanged
- rewrite only the project adapter's routing reference
